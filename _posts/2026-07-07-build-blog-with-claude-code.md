## 상황

개발하다 보면 AI와 문제를 풀면서 "아, 이거 배웠다" 하는 순간이 자주 온다.
그런데 그 순간을 기록하지 않으면 며칠 뒤엔 뭘 배웠는지도 잊어버린다.

그래서 목표를 세웠다 — **코딩 흐름을 끊지 않고, "이거 블로그에 올려줘" 한마디로
배운 것을 블로그 글로 남기는** 시스템을 만들자.

## 해결 과정

Claude Code와 기획부터 구현까지 함께 진행했다.

### 1. 제품이냐 스킬이냐부터 정리

가장 먼저 부딪힌 질문은 "이걸 거창한 제품으로 만들어야 하나?"였다.
결론은 **AI 에이전트 스킬**. 이유가 명확했다:

- 트리거("블로그에 올려줘")가 **AI와 코딩하는 세션 안에서** 발생한다.
  AI가 이미 대화 맥락을 다 쥐고 있으니, 맥락 추출이 공짜다.
- 1인 개인 도구라 무거운 파이프라인이 필요 없다.

### 2. 발행 플랫폼 선택 — API가 살아있는 곳

블로그 플랫폼을 고르다 의외의 벽을 만났다. 한국에서 많이 쓰는 곳들이
글쓰기 API를 막아둔 상태였다:

| 플랫폼 | 글쓰기 API |
|---|---|
| Velog | 공식 API 없음 (비공식 GraphQL만) |
| Tistory | 2024년 Open API 폐쇄 |
| Medium | 2023년 integration token 중단 |
| **GitHub Pages** | **git commit = 발행** |

결국 **GitHub Pages + Jekyll Chirpy**로 갔다. AI 에이전트가 git을
네이티브로 다루니, `.md` 파일 하나 커밋하는 게 곧 발행이다. 연동이 가장 단순하고,
기록이 온전히 내 소유로 남는다.

### 3. 블로그 세팅과 스킬 구현

- `chirpy-starter` 템플릿으로 레포를 만들고 한국어·서울 타임존으로 설정
- GitHub Pages를 **Actions 빌드**로 전환 (Chirpy는 legacy Jekyll 빌드로는 안 됨)
- 스킬 작성: 맥락 추출 → 초안 → **확인** → 커밋·push

### 4. Claude Code와 Codex, 둘 다 되게

Claude Code에서만 되면 아쉬웠다. Codex에서도 쓰고 싶었는데,
두 도구의 확장 방식이 달랐다:

- **Claude Code** — `~/.claude/skills/`에 두면 자연어로 자동 발동
- **Codex** — `~/.codex/prompts/`에 두면 `/blog` 슬래시 커맨드

그런데 **핵심 로직은 똑같은 마크다운 텍스트**다. 그래서 본문을 `instructions.md`
하나로 두고, 설치 스크립트가 두 도구용 래퍼로 감싸 각각 설치하게 했다.
단일 소스라 내용이 어긋날 일이 없다.

### 5. 남들도 쓰게 오픈소스로

내 블로그 경로가 하드코딩돼 있으면 남이 그대로 쓸 때 내 블로그로 커밋하려 든다.
그래서 경로·URL을 `~/.config/blog-publish/config.env`로 분리했다.
누구든 clone 후 `./install.sh`만 돌리면 자기 블로그를 가리킨다.
→ [github.com/sobokki/blog-publish-skill](https://github.com/sobokki/blog-publish-skill)

## 배운 것

- **AI 도구의 형태를 정할 때는 "트리거가 어디서 발생하는가"를 먼저 보자.**
  세션 안에서 일어나는 일이면 스킬이 맞다. 맥락이 이미 거기 있으니까.
- **외부 플랫폼 연동은 API 생존 여부부터 확인하자.** 멋진 UX를 설계해놓고
  API가 없어서 갈아엎는 것보다, 살아있는 API 기준으로 역산하는 게 빠르다.
- **Chirpy는 GitHub Pages 기본 legacy 빌드로는 안 뜬다.** `build_type`을
  `workflow`(GitHub Actions)로 바꿔야 테마가 제대로 빌드된다.
- **여러 도구를 지원할 땐 로직을 단일 소스로 두고 래퍼만 나누자.** Claude Code와
  Codex는 껍데기만 다르지 알맹이는 같았다. 중복은 곧 드리프트다.
- **하드코딩된 경로 하나가 "나만 쓰는 도구"와 "남도 쓰는 도구"를 가른다.** 설정 분리는
  공유의 최소 조건이다.
- 그리고 이 글 자체가, 방금 만든 그 스킬로 발행됐다. 도그푸딩 완료.
