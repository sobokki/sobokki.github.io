# Spec: AI 개발 성장 기록 블로그 + Claude Code 연동

작성일: 2026-07-06
상태: draft (구현 전)

## 1. 목적 (Why)

개발하다가 Claude Code / Codex 와 문제를 풀며 **새로 배운 개념·해결 과정**을,
그 순간 놓치지 않고 블로그로 흘려보낸다.

> "AI를 활용해 이런 기능을 구현했고, 이걸 배웠다" — 개발 성장 궤적을 기록으로 남긴다.

핵심 가치: 기록의 **마찰을 0에 가깝게**. 코딩 흐름을 끊지 않고 "이거 블로그에 올려줘" 한마디로 발행.

## 2. 사용자 (Who)

- 단일 사용자 (전소은 / sobokki)
- 개인 개발 성장 기록 목적. 팀 배포 제품 아님.

## 3. 시스템 구조

```
[어느 프로젝트든 Claude Code 세션 안]
   나: "이거 블로그에 올려줘"
        │  (전역 스킬 발동)
        ▼
   ┌─ 스킬 (~/.claude/skills/blog) ──────────┐
   │ 1. 대화 맥락에서 학습 포인트 추출          │
   │ 2. Chirpy .md 초안 작성                   │
   │    (제목/날짜/카테고리/태그/본문)          │
   │ 3. 초안 보여주고 확인 대기 ⏸              │
   │ 4. OK → 블로그 레포에 파일 저장            │
   │    → git commit → git push                │
   └──────────────────────────────────────────┘
        │  (GitHub Pages 자동 빌드 1~2분)
        ▼
   https://sobokki.github.io/posts/xxx   ✅ 발행
```

두 개의 조각:

- **조각 A — 블로그 껍데기**: `sobokki.github.io` 레포 + Jekyll Chirpy 테마 (1회 세팅)
- **조각 B — 연동 스킬**: `~/.claude/skills/blog/` 전역 스킬 (핵심)

연동 스킬은 어느 블로그 백엔드든 동일하게 작동한다 — 결국 `.md` 파일을 만들어 해당 레포에 커밋+push 하는 것.

## 4. 기능 요구사항 (Functional)

### 4.1 캡처 & 트리거
- [ ] 자연어로 발동: "블로그에 올려줘", "TIL로 기록해줘", "이거 블로그 글로 써줘"
- [ ] 현재 Claude Code 대화 맥락 전체를 입력으로 사용 (무슨 문제 → 어떻게 풀었나 → 뭘 배웠나)

### 4.2 초안 생성
- [ ] Chirpy frontmatter 자동 생성 (title, date, categories, tags)
- [ ] 한국어 본문. 기본 구조: `## 상황` → `## 해결 과정` → `## 배운 것`
- [ ] 카테고리·태그 자동 제안

### 4.3 검토
- [ ] 발행 전 초안 전체(.md)를 사용자에게 보여주고 확인 대기
- [ ] 사용자가 수정 요청하면 반영 후 재확인

### 4.4 발행
- [ ] `_posts/YYYY-MM-DD-slug.md` 형식으로 저장 (Chirpy 규칙 준수)
- [ ] `git add` → `commit` → `push`
- [ ] 발행될 URL 안내

## 5. 포스트 템플릿

```markdown
---
title: "Claude Code로 N+1 쿼리 잡은 이야기"
date: 2026-07-06 14:30:00 +0900
categories: [삽질기록, Backend]
tags: [claude-code, orm, performance]
---

## 상황
어떤 문제를 만났는지

## 해결 과정
Claude와 어떻게 접근했는지, 무엇을 시도했는지

## 배운 것
핵심 개념 / 다음에 써먹을 포인트
```

카테고리 초안 (Chirpy 규칙상 최대 2단계): `TIL` / `삽질기록` / `AI-협업` / `회고`

## 6. 기술 제약 (미리 확인된 것)

| # | 제약 | 대응 |
|---|---|---|
| 1 | 파일명은 반드시 `_posts/YYYY-MM-DD-slug.md` 형식 | 스킬이 규칙 강제. 안 지키면 렌더링 안 됨 |
| 2 | 한글 파일명은 URL 깨질 수 있음 | slug는 영문, title 프론트매터만 한글 |
| 3 | 날짜는 KST(+0900) 필요 | 스킬이 `date` 명령으로 현재 시각 생성 |
| 4 | 블로그 레포 경로를 스킬이 알아야 함 | 고정 경로 `/Users/soeun/sobokki.github.io` (cwd 아님) |

플랫폼 API 제약 (기획 단계에서 확인):
- Velog: 공식 글쓰기 API 없음 (비공식 GraphQL, 불안정) → 제외
- Tistory: Open API 2024 폐쇄 → 제외
- Medium: integration token 2023 중단 → 제외
- **GitHub Pages: git commit = 발행. Claude Code가 git 네이티브라 연동 최적** → 채택

## 7. 범위 (Scope)

**In scope (v1)**
- Chirpy 블로그 세팅 + GitHub Pages 배포
- 대화 맥락 → 한글 포스트 초안 생성
- 초안 확인 → 커밋/푸시 발행
- 카테고리/태그 자동 제안

**Out of scope (나중에)**
- 이미지/스크린샷 자동 첨부
- 기존 글 수정/삭제
- 여러 플랫폼 동시 발행 (velog 크로스포스팅 등)
- 초안 없이 자동 즉시 발행

## 8. 구현 계획 (Phase)

| Phase | 내용 | 완료 기준 |
|---|---|---|
| **1. 블로그 껍데기** | `sobokki.github.io` 레포 생성, Chirpy 설치, 프로필/카테고리 설정, GitHub Pages 배포 | 브라우저에서 접속되는 블로그 URL |
| **2. 연동 스킬** | `~/.claude/skills/blog/SKILL.md` 작성 (맥락추출→초안→확인→발행) | 스킬 발동 → 초안 생성 확인 |
| **3. 검증 (도그푸딩)** | 이 대화 자체를 첫 포스트로 발행 | 첫 글이 실제 URL에 뜸 |

## 9. 인수 조건 (Acceptance Criteria)

- [ ] 어느 프로젝트에서 코딩하든 "블로그에 올려줘"로 스킬이 발동한다
- [ ] 스킬이 대화 맥락에서 상황/해결/배운점을 뽑아 한글 초안을 만든다
- [ ] 발행 전 초안을 보여주고 내 확인을 기다린다
- [ ] 확인하면 올바른 파일명 형식으로 커밋·push되고, 1~2분 뒤 블로그에 뜬다
- [ ] 발행될 URL을 알려준다
